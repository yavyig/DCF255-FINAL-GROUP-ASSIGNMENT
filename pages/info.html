Background of IMAP

  • IMAP stands for Internet Message Access Protocol.
  • Open standard that describes how to access messages in an email mailbox.
  • The current version is IMAP4 even though most interface don’t include the number
  • Protocols used for reading emails, and retrieving emails from an email server so you can access to them on your own devices such as on your laptop/desktop/mobile, etc.
  • IMAP is a more effectively way of accessing email since it acts like a form of cloud storage compare to POP.
  • A more complex protocol than POP.
  • It leaves emails on the server while caching meaning temporarily storing emails locally.


How does IMAP Works

IMAP is an interactive client-server model, it connects to the email server and clients can send their request such as reading a specific emails to the server. This request is then cached locally meaning that clients are not downloading any local copies of the message but simply can work on their own devices. Once they have made changes to their emails, such as deleting or sending a new email, the server will process and save the changes before disconnecting. IMAP can manipulate persistent message status flags which includes “Seen”, “Deleted”, “Answered”, as well as the user-defined flags. IMAP allows concurrent updates and access to shared mailboxes. This is effectively when multiple clients are processing messages for a common inbox. Changes can be seen to all the concurrently actives clients through this protocol. IMAP allows defer transferring of message which is useful when is connecting to a mail server through low band-with lines. In another words, client can choose to read certain part of the message instead of retrieving everything all at once to save more time and be more efficient. However, there are freely available IMAP development libraries, and is a protocol that is more complex than POP. This would not be a disadvantage though because it is changing very rapidly, and development is being more advanced now to further develop and improve on this protocol.




Advantages of IMAP

There are many advantages of IMAP and that is why it is more commonly used among clients.

  1. Mail is stored on the remote server so it is effective and accessible from several devices.
  2. All changes are saved and tracked on the server, prevent duplicate inboxes.
  3. Allows faster overview because only headers are displayed until content is explicitly requested.
  4. If the sever is managed properly, the mails are automatically backed up and saved.
  5. IMAP saves local storage space because will not download anything unless you requested from your devic.
  6. Also allows the option of downloading and storing mails locally if needed.
  7. Allows multiple clients to manage the same inbox which is in line with how people use email nowadays.
  8. Even though internet connection is needed to access mail, but it is also possible to work office and sync or save changes when. You are back online.


The Disadvantage of IMAP

Most of the email providers only allows limited amount of space. Therefore, client will have to clean out their email often depending on their workload and if they have lots of messages in their accounts.


Common IMAP Extensions

  • IDLE – allows real-time notification of received mail.
  • SORT – will arrange message at the server so that the email account can fetch certain ones without downloading all of them.
  • THREAD – clients can retrieve related messages instead of downloading all mail in a folder
  • CHILDREN – implements a hierarchy of folders.
  • ACL = Access Control List – individual rights is specified in each IMAP folder.
